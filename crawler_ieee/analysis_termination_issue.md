# IEEE 크롤링 조기 종료 원인 분석 보고서

## 1. 문제 현상
*   **실행 명령어**: `python crawling_ieee_2023_2025.py --headless --journal-option all --years all --resume`
*   **현상**: 페이지 210까지 완료 후, 페이지 211에서 "검색 결과 0건"이 감지되어 크롤링이 종료됨.
*   **사용자 의문**: 저널 설정이 되어 있는데 왜 벌써 끝났는지, 저널을 하나씩 다 돌지 않은 것인지?

## 2. 분석 결과

### ① `--journal-option all` 옵션의 동작 방식
*   코드 분석 결과, `--journal-option all` 옵션은 **"Publication Title 필터(저널 필터)를 전혀 사용하지 않고 전체 검색"**하는 모드입니다.
*   이 옵션을 사용하면 `JOURNAL_TARGETS` 리스트(TGRS, GRSL 등)를 하나씩 순회하는 루프를 **건너뛰게 됩니다.**
*   대신, IEEE Xplore에서 아무런 필터 없이(또는 연도 필터만 가지고) 검색을 수행한 뒤, 그 결과 전체를 페이지별로 다운로드하려고 시도합니다.

### ② IEEE Xplore의 페이징 제한 (Paging Limit)
*   IEEE Xplore는 검색 결과가 아무리 많더라도 UI상에서 넘겨볼 수 있는 검색 결과 수를 **약 2,000~2,100건(또는 100~200 페이지)** 내외로 제한하고 있습니다.
*   현재 스크립트는 페이지당 10개 항목(`Items per page: 10`)으로 설정되어 있습니다.
*   따라서 페이지 210(약 2,100건) 근처에 도달하면 IEEE Xplore가 더 이상 다음 결과를 보여주지 않게 되며, 스크립트는 이를 "모든 결과 수집 완료" 또는 "마지막 페이지 초과"로 인식하여 정상 종료하게 된 것입니다.

### ③ `--years all` 옵션의 영향
*   `--years all` 역시 특정 연도 범위를 지정하지 않고 전체 연도를 검색하게 하므로, 검색 결과가 너무 방대해져서 위에서 언급한 페이징 제한에 더 빨리 걸리게 됩니다.

## 3. 결론 및 권장 사항

사용자께서 의도하신 **"여러 저널을 하나씩 순회하며 모든 논문을 다운로드"**하려면 옵션 설정을 변경해야 합니다.

### 권장 실행 명령어
저널 리스트를 순차적으로 돌면서 수집하려면 `--journal-option all`을 **제거**해야 합니다.

```bash
# 저널 리스트 50개를 순차적으로 크롤링 (연도별로 나누어 실행 권장)
python crawling_ieee_2023_2025.py --headless --years 2023 2024 2025 --num-journals 50 --resume
```

### 상세 가이드
1.  **`--journal-option all` 사용 지양**: 이 옵션은 특정 키워드 없이 "최근 논문 2,000건" 정도를 빠르게 훑을 때나 유효하며, 전체 저널 데이터를 수집하기에는 부적절합니다.
2.  **저널 순회 모드 활용**: `--journal-option`을 생략하면 스크립트 내부에 정의된 `JOURNAL_TARGETS_ALL` (50개 저널/학회) 리스트를 하나씩 방문하며 필터를 적용해 크롤링합니다. 이 방식은 각 저널별로 결과가 2,000건을 넘지 않는 경우가 많아 페이징 제한 문제를 피할 수 있습니다.
3.  **`--num-journals 50` 설정**: 기본값이 30개로 설정되어 있을 수 있으므로, 준비된 50개 저널을 모두 돌려면 이 옵션을 명시적으로 주는 것이 좋습니다.
4.  **연도별 실행**: `--years all` 보다는 `--years 2023 2024 2025`와 같이 구체적인 연도를 나열하는 것이 더 안정적입니다.
